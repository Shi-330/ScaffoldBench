# ScaffoldBench
> **We don't test models. We test the scaffold.**
>
> 如果有人给模型打分，为什么不也给手脚架打分呢？

市面上 AI Agent 并没有将 AI 模型和 智能体的部分 拆分开, 我们如何衡量到底是模型的能力变强了还是手脚架给了强大的助力。

**ScaffoldBench 想要建立一套评价，就是脱离模型，手脚架的水平如何，给工程策略跑分。**

## 怎么测

```
固定弱模型/中等水平模型 + 不同手脚架 + 同一套任务 = 手脚架的含金量
```

### 三个核心指标

| 指标 | 测什么 | 怎么测 |
|------|--------|--------|
| **Strategy Score** | 工程策略含金量 | 强制接入 Llama-3-8B 级别的开源小模型，看手脚架能不能让弱模型表现力更好 |
| **Token Efficiency** | 经济效能比 | `Task Pass Rate ÷ Total Tokens`，最少的token达成最多的任务，追求性价比。 |
| **Fault Recovery** | 抗脆弱性 | 故意在工具调用中注入错误返回，测异常捕获与自我纠偏能力 |

### 控制变量

- 同一个开源小模型（默认 Qwen-2.5-7B / Llama-3-8B）
- 同一套标准任务集（预想是使用SWE-bench，但是也有可能对弱智模型，分辨率太粗无法评估）
- 相同的运行环境（Docker）
- 也许手脚架是一种AI平权的方式，让好的工程学设计配上普通的模型，获得更具性价比的能力，让token成为下一个4g网络。

## 项目状态

这只是一个很早期的想法。欢迎交流，欢迎提交手脚架来跑分。
下一步想先做一个垂直场景验证方法论，比如说多步骤文件操作，任务集围绕这个场景设计，跑通之后再考虑扩展（依旧，想做上文预想的SWE-bench）。
